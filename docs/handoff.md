### あらすじ探索の基準（2026-07-08に確立・`docs/operations.md`⑧に反映済み）
- 公式トップページで「記載なし」と即断せず、「物語」「STORY」「INTRODUCTION」「作品概要」等の
  **個別サブページ**を探す（WebFetchに「リンク一覧を教えて」と聞くと見つかる）
- JS描画（SPA）サイトは `https://r.jina.ai/<URL>` を頭に付けて再取得
- ストーリー本文が**画像埋め込み**の場合はテキスト抽出不可＝取得不可として見送る
- 公式・dアニメどちらも無ければ、`eiga.com`（解説・あらすじ欄）・`animatetimes.com`（ニュース記事）を
  第3候補として使ってよい（2026-07-08にユーザー承認）。Wikipedia・二次創作まとめは引き続き禁止
- エピソード単位のあらすじしか無い場合は、それを作品紹介として使ってよい

### 作業上の注意（過去セッションで踏んだ落とし穴）
- **dev server 起動中に `rm -rf .next && npm run build` を実行すると dev server のキャッシュが壊れる**（`Cannot find module './xxx.js'`）。ビルド前に `preview_stop` してから。ビルド後に dev を使うなら再起動する。
- ポート3000が他セッションで使用中のことがある。`.claude/launch.json` は `autoPort: true` を追加済み（別ポートで自動起動する）。
- コミット前の定番チェック: `npx tsc --noEmit` → `node scripts/check.ts`（37/37）→ `preview_stop` → `rm -rf .next && npm run build`。feed.xml のビルドが稀にAnnict API瞬断で失敗するが、再ビルドで通る。
- AIサムネのプロンプトは「no characters」だけだと人物が入り込むことがある。`absolutely no people, no figures, no silhouettes` のように強めに否定する。生成後は必ず `Read` で1枚ずつ目視確認。
