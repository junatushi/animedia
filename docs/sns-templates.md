# SNS投稿テンプレート集

`.github/workflows/post-to-x.yml` の「Run workflow」実行時、`message` 欄にそのままコピペして使う想定の下書き集。
`{URL}` は実際のリンクに置き換える（季節ページのディープリンクにすると、開いた人が同じ内容を見られる。例: `https://animedia-khaki.vercel.app/?year=2026&season=summer`）。

---

## 0. 自動生成コマンド（2026-07-13導入）

以下は下の1〜4のテンプレを手で埋める代わりに、実データから自動で文案を作るショートカット。
`node scripts/print-digest.js` にコマンドライン引数を渡すだけで、コピペ用の文面が標準出力に出る
（`.github/workflows/`のCI専用ではなく、手元で今すぐ使いたい時にも使える）。

```bash
# 2.「配信情報が埋まってきた報告」を自動生成（件数は/api/seasonから実測、手で数えない）
node scripts/print-digest.js coverage

# 4.「新機能の告知」を自動生成（機能名・説明は自分で渡す＝一次情報のまま、文章の創作はしない）
node scripts/print-digest.js feature "独占チップ" "配信サービスをAND条件で絞れるようになり、「dアニメの独占配信」のような検索ができます。"
```

- `coverage`: `lib/services.ts`のAnimeItem.servicesが1件以上ある作品数／全体件数を都度APIから取得して埋め込む。数字を手で確認・入力し直す手間が要らない
- `feature`: 機能名・説明を渡すとテンプレA（「▶ 機能名」＋説明＋URL）の形に整形するだけ。説明文自体はこちらが書いた事実をそのまま使う（自動で誇張・創作はしない）
- 出力はXへの手動投稿用の下書き。Bluesky/Mastodon個別の文体調整をしたい場合はここから手直しする
- 1.新シーズン開始の告知／3.バグ修正の告知は、機能名・件数のような差し込みデータが薄いため、
  引き続き下記テンプレを手直しする運用のままにしている

## 1. 新シーズン開始の告知

**A. 端的**
```
2026年夏アニメの配信情報、まとめました。
サービス別に絞り込み・お気に入り登録もできます。
{URL}
```

**B. ちょっと煽り気味**
```
今期どこで見れるか、探すの面倒じゃないですか？
dアニメ・ABEMA・Netflix…配信サービス別に一発で絞り込めるサイト作りました。
{URL}
#2026夏アニメ
```

**C. 機能訴求**
```
アニメ視聴ガイド、今期分を更新しました。
・配信サービスで絞り込み（AND/OR切替可）
・お気に入り登録（ログイン不要）
・人気順/五十音順の並び替え
{URL}
```

## 2. 配信情報が埋まってきた報告（シーズン開始2〜3週間後）

```
Annict側の登録が進み、だいぶ埋まってきました。
まだの作品もあるので随時反映されます。
{URL}
```

**B. 具体的な数字入り**

```
今期アニメの配信情報、現在◯件中◯件で配信サービスが判明しています。

```

**A. 軽く**

## 4. 新機能の告知

**A. 汎用テンプレート**
```
アニメ視聴ガイドに新機能を追加しました。
▶ {機能名: 例 お気に入り登録機能}
{機能の一言説明}
{URL}
```

**B. お気に入り機能の例**
```
「今期どれ追ってたか忘れる」を解消。
ログイン不要のお気に入り登録機能を追加しました。
★ボタンを押すだけ、ブラウザに保存されます。
{URL}
```

**C. 並び替え/絞り込みの例**
```
配信サービスを2つ以上選ぶと「AND/OR」を切り替えられるようにしました。
「dアニメでもABEMAでも見れる作品」「dアニメとABEMA両方にある作品」を絞り込めます。
{URL}
```

## 5. 毎日投稿用ローテーション（2026-07-07導入 / 2026-07-08にBluesky・Mastodonは自動化）

曜日ごとに「何を書くか」を固定した7日サイクル。

**Bluesky・Mastodonは`daily-digest.yml`が毎日この型で自動投稿する**（下記「日〜土」の内容を
`build-digest.js`が自動生成し、サイトのカレンダー画面/TOP5パネルのスクリーンショットを
自動で添付する。2026-07-14導入。詳細は`docs/operations.md`の「⑦毎日ダイジェスト」参照）。
以下の型は、**Xの手動投稿**や、自動文面を手直ししたいときの参考。すべて数字・作品名は
投稿直前に実サイトで確認する。

**月〜土: 「今日配信のアニメ」（カレンダー機能と連動）**
```
【{曜日}曜日の配信情報】
本日配信/放送の今期アニメはこちら。
{URL}
```
- `{URL}?view=calendar&day={曜日}`（例: `&day=火`）でカレンダー表示・該当曜日タブを直接開ける
  （2026-07-14導入）。Xに手動投稿する際はこの画面を手元でスクリーンショットして添付する。
- 気が向けば、その日の作品を1〜2本ピックアップして本文に追記する
  （例: `中でも「{作品名}」が{配信サービス}で今日配信。`）。

**日曜: 「今期の注目作TOP5」＋その日の配信アニメ紹介**
```
今週の「アニメ視聴ガイド」注目作TOP5（{年}年{季節}アニメ）
1. {作品名}（{watchers}人が注目）
...
{URL}
```
- `{URL}?ranking=open`で「今期の注目作TOP5」パネルを開いた画面を直接開ける（2026-07-14導入）。
  Xに手動投稿する際はこの画面をスクリーンショットして添付する。
- TOP5は`watchers`降順で変動するため、投稿前に実際のTOP5を確認してから選ぶ。

**ネタが尽きた日の予備（配信サービススポットライト）**
```
今期、{配信サービス名}で見られるアニメをまとめてチェックできます。
{URL}
```
- サービス絞り込みチップを押した状態のURL（例: `?service=danime` のようなクエリは
  現状未対応。「サイトでdアニメのチップを押すと絞り込めます」と一言添える）。

## 運用メモ（2026-07-08時点の実態に更新）

**現在の自動化状況**（`.github/workflows/weekly-digest.yml`、毎週月曜21:00 JST自動実行）:
- **Bluesky・Mastodonへは自動投稿されている**（Secrets登録済み・動作確認済み）。何もしなくてよい
- **Xだけは手動**。ワークフローが「今期TOP5」の下書きを[GitHub Issue](https://github.com/junatushi/animedia/issues)として自動起票するので、
  1. Issuesタブで最新の「Xへの投稿下書き (日付)」を開く
  2. 本文をコピー
  3. X（ブラウザ/アプリ）から自分で投稿
  という流れになる（X APIが従量課金化したため、API経由の自動投稿はしない）

**`post-to-x.yml`（「Xへ手動投稿」ワークフロー）は使わない**: これはX API経由でActions上から直接投稿するための古い仕組みで、`X_API_KEY`等のSecretsは課金回避のため意図的に未登録にしてある。実行しても失敗するだけで、そもそも使う想定ではない（存在するのはコードの記録として）。Xへの投稿は必ず上記の「Issue下書きをコピーしてブラウザから手動投稿」で行うこと。

- ハッシュタグは季節もの（`#2026夏アニメ` 等）以外は基本つけない方針でシンプルに保つ。
- 数字（件数等）を入れる場合は投稿直前に実際のサイトで確認してから入力する（古い数字を貼らない）。
