# X成長プレイブック（フォロワー獲得の週次サイクル・2026-07-16導入）

「アニメ視聴ガイド」のX（旧Twitter）アカウントを、興味を持ってくれそうなユーザーに
リーチしてフォロワーを増やすための運用手順。**仕組みは半自動・実行は手動**が基本方針。

---

## 前提（なぜ「全自動」ではないのか）

- X APIは2026年2月に従量課金化（投稿1件$0.015〜$0.20、検索も有料枠）。自動投稿・自動検索は
  費用対効果が合わない（`docs/sns-growth-research.md`）。
- 自動フォロー/フォロー解除・大量DM・スクレイピングは**XのToS違反**でアカウント凍結リスクが
  ある。**やらない**。
- よって成長エンジンは「**毎週の手動アクションの材料を自動でそろえる**」ところまでを担い、
  実際の投稿・リプ・フォローは人が行う。数字・作品名・配信先は必ずサイトの実データ
  （`/api/season`＝Annict）から出し、推測・創作しない。

## フォロワーが増える3つの経路（この運用が狙うもの）

1. **一貫した価値ある発信** … 日次ダイジェスト（`daily-digest.yml`）で毎日カバー済み。
2. **リーチ（会話への参加）** … 「配信どこ？」で困っている人・今期アニメを話している人に
   自然に絡む。← **週次成長キットが新たに仕組み化した部分**。
3. **フォロー転換** … プロフィール訪問者を「フォローする理由」に変える。固定ポスト（反応の
   良かった投稿）とプロフィールの整備がここに効く。

---

## 週次サイクル（毎週月曜にIssueが自動起票される）

`.github/workflows/x-growth.yml` が**毎週月曜 20:00 JST**に実行し、「今週のX成長アクション」を
GitHub Issueとして起票する（本文は `scripts/print-growth-kit.js` →
`scripts/lib/build-growth-kit.js`）。Issueを開いて上から順にこなす:

1. **投稿ドラフト（4本）** … 独占配信ピック/配信サービス別ピック/声優ピック/データもの。
   日次ダイジェストとは切り口を変えてある。コピペして2〜3本投稿する。各ドラフトのリンク先は
   サイトのSEOページ（`/exclusive` `/service` `/rankings` `/person`）で、内部リンク強化も兼ねる。
2. **リーチ** … 記載の検索クエリを**Xの検索窓にコピペ**して見込み客を探し、リプ下書きを添えて
   自然に返す。いきなりURL直貼りはせず、まず会話→役立つ場面で1回だけ貼る。
3. **投稿時刻・ハッシュタグ** … 21:00〜24:00 か 昼12時台。タグは付けすぎない。
4. **週次チェック** … 投稿/リプ/フォロー/固定ポスト候補メモの4項目にチェック。

> 手元で内容を先に見たいときは `node scripts/print-growth-kit.js` を実行すればMarkdownが出る。
> 手動でIssueを今すぐ作りたいときは GitHub の Actions →「週次X成長キット」→ Run workflow。

## 固定ポスト・プロフィール（手動・随時）

キットには含めていない（あなたのXアカウント側で手動反映する領域のため）。ただしフォロー転換に
直結するので、四半期に1回は見直す:
- **固定ポスト**: 直近で最も反応の良かった投稿を固定する（キットの週次チェックでメモした候補から）。
- **プロフィール**: 「今期アニメがどこで配信されているか一覧でわかるサイトの中の人」と一目で
  伝わる説明＋サイトURL。

## 日次ダイジェストとの役割分担

| | 日次ダイジェスト（`daily-digest.yml`） | 週次成長キット（`x-growth.yml`） |
|---|---|---|
| 頻度 | 毎日 21:00 JST | 毎週月曜 20:00 JST |
| 中身 | その日の放送/配信作品・TOP5 | 投稿ネタ4本＋リーチ用クエリ/リプ＋チェックリスト |
| 狙い | 継続発信（経路1） | リーチ＋フォロー転換（経路2・3） |
| 投稿先 | Bluesky/Mastodon自動＋X下書きIssue | X向けの手動アクションIssueのみ |

## 頻度・トーンの調整

- 反応が悪い・スパム的に見えると感じたら、`x-growth.yml` の cron を隔週
  （`0 11 * * 1` を2週に1回相当へ）などに間引く。日次ダイジェスト側も同様に間引ける
  （`docs/operations.md`「⑦」）。
- リプは営業色を出しすぎない。まず相手の話に乗る→本当に役立つ時だけリンクを出す。

## 効果測定

- どの投稿・作品・サービスが響いたかは `/admin/analytics`（`share_work`等）と、X側の
  インプレッション/プロフィールアクセスで見る。伸びたネタの切り口をキットの定番に育てる。
- 生成AI検索での引用は別途 週次モニタリング（`docs/operations.md`「⑨」）。
